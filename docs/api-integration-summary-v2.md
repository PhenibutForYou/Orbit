# Сводка по API и конфигурации фронтенда

Актуальная упрощенная версия контракта мониторинга. Фронтенд получает стартовый snapshot объектов, затем слушает события `created`, `updated`, `deleted`.

## Главная идея

Фронт только отображает данные. Он не вычисляет аварийность, не определяет состояние параметров и не решает, когда объект потерял связь.

Бэкенд присылает:

- общий статус объекта;
- координаты объекта;
- статические параметры объекта;
- список параметров телеметрии при создании;
- короткие обновления телеметрии в рантайме.

## Стартовый запрос

```http
GET /api/objects
```

Ответ:

```json
{
  "objects": [],
  "maxObjects": 15
}
```

Если на сервере уже есть подключенные объекты, они должны прийти сразу в `objects`. Если объектов нет, приходит пустой массив.

## Формат объекта

```json
{
  "id": "car-01",
  "type": "car",
  "name": "Машина-01",
  "description": "Сервисный автомобиль патрульной группы",
  "status": "normal",
  "coordinates": "-48.42152, 37.88403",
  "updatedAt": "2026-07-02T12:45:31.000Z",
  "static": [
    { "label": "Точка начала", "value": "-71.42015, 62.14577" },
    { "label": "Точка прибытия", "value": "64.30184, -32.86719" }
  ],
  "telemetry": [
    {
      "key": "fuel",
      "label": "Топливо",
      "progress": 72,
      "value": "72 %",
      "status": "normal"
    }
  ]
}
```

Обязательные поля:

- `id` - уникальный идентификатор.
- `type` - машинный тип объекта: `car`, `warehouse`, `fuel-station`, `drone`.
- `name` - название объекта.
- `description` - описание объекта.
- `status` - общий статус объекта.
- `coordinates` - координаты объекта в условной системе карты.
- `updatedAt` - ISO-время последнего обновления объекта.
- `static` - статические параметры объекта.
- `telemetry` - динамические параметры объекта.


## Координаты

Координаты не географические, а условные декартовы:

- центр карты - `0, 0`;
- диапазон X - от `-100` до `100`;
- диапазон Y - от `-100` до `100`;
- формат - `"x, y"`;
- дробные значения ожидаются, например `"51.42152, -17.62080"`.

Любой объект может изменить координаты в любой момент. Поля `movable` нет.

## Статусы

Допустимые значения:

- `normal` - норма, зеленый цвет.
- `warning` - предупреждение, желтый цвет.
- `alert` - критическое состояние, красный цвет.
- `nodata` - нет данных, серый цвет.

Статус объекта приходит в `object.status`.

Статус параметра приходит в `telemetry[].status` или `telemetryUpdates[].status`.

## Статические параметры

Статические параметры передаются в поле `static`.

```json
[
  { "label": "Точка начала", "value": "-71.42015, 62.14577" },
  { "label": "Точка прибытия", "value": "64.30184, -32.86719" }
]
```

Точки начала/прибытия считаются обычными статическими параметрами. На карте они отдельно не отображаются.

## Телеметрия при создании объекта

При создании или в стартовом snapshot параметр телеметрии должен содержать `key` и `label`, потому что фронт должен знать, как подписать строку.

```json
{
  "key": "charge",
  "label": "Заряд",
  "progress": 81,
  "value": "81 %",
  "status": "normal"
}
```

Поля:

- `key` - стабильный ключ параметра внутри объекта.
- `label` - подпись параметра в интерфейсе.
- `progress` - число от `0` до `100` для визуального прогрессбара.
- `value` - готовый текст для отображения.
- `status` - статус конкретного параметра.

`unit`, `range`, `rawValue` фронту сейчас не нужны.

## Телеметрия при обновлении

В update-событии параметру уже не нужен `label`. Фронт знает подпись по `key`, потому что она пришла при создании объекта.

Минимальный формат:

```json
{
  "key": "charge",
  "progress": 28,
  "value": "28 %",
  "status": "warning"
}
```

Это достаточно для текущего UI:

- `key` - найти нужную строку;
- `progress` - плавно изменить прогрессбар;
- `value` - заменить текстовое значение;
- `status` - перекрасить параметр.

## Событие `created`

```json
{
  "type": "created",
  "object": {
    "id": "drone-05",
    "type": "drone",
    "name": "Дрон-05",
    "description": "БПЛА наблюдения периметра",
    "status": "normal",
    "coordinates": "12.50124, -61.90421",
    "updatedAt": "2026-07-02T12:45:36.000Z",
    "static": [
      { "label": "Точка начала", "value": "-36.50711, -50.91033" },
      { "label": "Точка прибытия", "value": "78.21084, 44.90873" }
    ],
    "telemetry": [
      {
        "key": "charge",
        "label": "Заряд",
        "progress": 81,
        "value": "81 %",
        "status": "normal"
      }
    ]
  }
}
```

После `created` фронт проигрывает анимацию появления объекта в списке и на карте.

## Событие `updated`

```json
{
  "type": "updated",
  "objectId": "drone-05",
  "patch": {
    "coordinates": "14.88441, -59.12005",
    "status": "warning",
    "updatedAt": "2026-07-02T12:45:39.000Z",
    "telemetryUpdates": [
      {
        "key": "charge",
        "progress": 28,
        "value": "28 %",
        "status": "warning"
      }
    ]
  }
}
```

`patch` может содержать только изменившиеся поля.

Поддерживаемые поля:

- `coordinates` - фронт плавно двигает маркер.
- `status` - фронт меняет цвет объекта.
- `updatedAt` - фронт обновляет время.
- `telemetryUpdates` - фронт обновляет только указанные параметры.
- `static` - можно передать, если статические параметры объекта реально изменились.

## Событие `deleted`

```json
{
  "type": "deleted",
  "objectId": "drone-05"
}
```

После `deleted` фронт проигрывает анимацию удаления. Если объект был выбран, выделение снимается.

## Ограничение количества объектов

Фронт отображает максимум `maxObjects`.

```json
{
  "maxObjects": 15
}
```

Если backend не вернул `maxObjects`, используется фронтовой fallback:

```env
VITE_MAX_OBJECTS_COUNT=15
```

## Frontend API слой

Файл `src/api/monitoringApi.js` является адаптером реального backend API.
Он загружает стартовый snapshot через `GET /api/objects` и подписывается на события через `VITE_API_EVENTS_ENDPOINT`.

Файл `src/api/httpClient.js` содержит общий HTTP-клиент с timeout, сборкой URL и обработкой ошибок.

Файл `src/api/historyApi.js` работает с `/api/history` и `/api/history/export`.

## Что фронт считает сам

Фронт сам считает только отображение:

- русскую подпись статуса;
- относительное время;
- позицию маркера на карте из координат;
- визуальные анимации.

Фронт не считает аварийность и не определяет итоговый статус объекта.

## Можно ли упростить еще

Да, потенциально можно:

- убрать `description`, если оно не нужно в правой панели;
- не слать `maxObjects` с бэка, если лимит будет только фронтовым конфигом;
- переименовать `telemetryUpdates` в короткое `telemetry`, если бэкендеру так удобнее, но сейчас `telemetryUpdates` яснее показывает, что это частичный patch;
Но прямо сейчас я бы оставил `description`, `maxObjects` и `telemetryUpdates`: они не перегружают контракт, зато делают интеграцию понятнее и устойчивее.

## Архив

Архив пока не входит в monitoring API. Планируемые endpoint-ы:

```http
GET /api/history
GET /api/history/export
```
