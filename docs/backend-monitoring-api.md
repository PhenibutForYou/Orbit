# Backend API мониторинга

> Актуальная упрощенная версия контракта находится в `docs/api-integration-summary-v2.md`.

Документ описывает контракт для подключения реального бэкенда к текущему фронтенду мониторинга. Архив пока не входит в этот контракт.

## Общая схема

Фронтенд работает в два этапа:

1. При открытии страницы делает стартовый запрос и получает полный снимок уже подключенных объектов.
2. После этого слушает события бэкенда: создание, обновление и удаление объектов.

Транспорт событий можно выбрать любой: WebSocket, SSE или polling. Для фронтенда важен формат данных.

## GET `/api/objects`

Возвращает все объекты, которые уже есть на сервере на момент открытия сайта.

```json
{
  "objects": [
    {
      "id": "car-01",
      "type": "car",
      "name": "Машина-01",
      "description": "Сервисный автомобиль патрульной группы",
      "status": "normal",
      "coordinates": "-48.42152, 37.88403",
      "updatedAt": "2026-07-02T12:45:31.000Z",
      "static": [
        { "label": "Точка начала", "value": "-71.42015, 62.14577" },
        { "label": "Точка прибытия", "value": "64.30184, -32.86719" }
      ],
      "telemetry": [
        {
          "key": "fuel",
          "label": "Топливо",
          "value": "72 %",
          "rawValue": 72,
          "unit": "%",
          "range": { "min": 0, "max": 100 },
          "progress": 72,
          "status": "normal",
          "updatedAt": "2026-07-02T12:45:31.000Z"
        }
      ]
    }
  ],
  "maxObjects": 15
}
```

## Объект

Обязательные поля:

- `id` - уникальный идентификатор объекта.
- `type` - машинный тип объекта: `car`, `warehouse`, `fuel-station`, `drone`.
- `name` - название объекта.
- `description` - описание объекта.
- `status` - общий статус объекта.
- `coordinates` - координаты в условной системе карты от `-100` до `100`, формат `"x, y"`.
- `updatedAt` - время последнего обновления объекта в ISO 8601.
- `static` - статические параметры, приходят при snapshot или создании объекта.
- `telemetry` - текущие динамические параметры с диапазонами и статусами.

## Статусы

Фронтенд не вычисляет итоговый статус сам. Бэкенд присылает общий статус объекта и статус каждого параметра телеметрии.

Допустимые значения:

- `normal` - норма, зеленый цвет.
- `warning` - предупреждение, желтый цвет.
- `alert` - критическое состояние, красный цвет.
- `nodata` - нет данных, серый цвет.

Если долго нет данных от сущности, это должен обработать бэкенд и прислать `status: "nodata"`.

## Телеметрия

Каждый параметр телеметрии должен иметь стабильный `key`. По нему фронтенд понимает, какой параметр обновился.

```json
{
  "key": "charge",
  "label": "Заряд",
  "value": "28 %",
  "rawValue": 28,
  "unit": "%",
  "range": { "min": 0, "max": 100 },
  "progress": 28,
  "status": "warning",
  "updatedAt": "2026-07-02T12:45:39.000Z"
}
```

Поля:

- `key` - стабильный ключ параметра внутри объекта.
- `label` - название параметра для интерфейса.
- `value` - готовое отображаемое значение.
- `rawValue` - числовое значение для логики и графиков.
- `unit` - единица измерения.
- `range` - допустимый диапазон.
- `progress` - значение для шкалы в процентах от `0` до `100`.
- `status` - цветовое состояние конкретного параметра.
- `updatedAt` - время обновления параметра.

## Событие `created`

Присылается, когда на сервере появился новый объект.

```json
{
  "type": "created",
  "object": {
    "id": "drone-05",
    "type": "drone",
    "name": "Дрон-05",
    "description": "БПЛА наблюдения периметра",
    "status": "normal",
    "coordinates": "12.50124, -61.90421",
    "updatedAt": "2026-07-02T12:45:36.000Z",
    "static": [
      { "label": "Точка начала", "value": "-36.50711, -50.91033" },
      { "label": "Точка прибытия", "value": "78.21084, 44.90873" }
    ],
    "telemetry": []
  }
}
```

После `created` фронтенд проигрывает анимацию появления объекта в списке и на карте.

## Событие `updated`

Присылается, когда изменились координаты, общий статус или телеметрия объекта.

Событие может содержать только изменившиеся поля. Для телеметрии используется `telemetryUpdates`.

```json
{
  "type": "updated",
  "objectId": "drone-05",
  "patch": {
    "coordinates": "14.88441, -59.12005",
    "status": "warning",
    "updatedAt": "2026-07-02T12:45:39.000Z",
    "telemetryUpdates": [
      {
        "key": "charge",
        "label": "Заряд",
        "value": "28 %",
        "rawValue": 28,
        "unit": "%",
        "range": { "min": 0, "max": 100 },
        "progress": 28,
        "status": "warning",
        "updatedAt": "2026-07-02T12:45:39.000Z"
      }
    ]
  }
}
```

Важно:

- Если изменились координаты, фронтенд плавно перемещает маркер.
- Если изменился общий статус, фронтенд меняет цвет объекта.
- Если изменился параметр телеметрии, фронтенд подсвечивает только этот параметр.
- При простом клике по объекту подсветка телеметрии не запускается, потому что это не новое событие от бэкенда.

## Событие `deleted`

Присылается, когда объект удален на сервере.

```json
{
  "type": "deleted",
  "objectId": "drone-05"
}
```

После `deleted` фронтенд проигрывает анимацию удаления. Если удаленный объект был выбран, выделение снимается.

## Координаты

Координаты остаются в условной декартовой системе текущей карты:

- центр карты - `0, 0`;
- диапазон по каждой оси - от `-100` до `100`;
- формат строки - `"x, y"`;
- дробные значения допустимы и ожидаются, например `"51.42152, -17.62080"`.

Статические точки маршрута, например `Точка начала` и `Точка прибытия`, передаются в `static` и визуально на карте пока не отображаются.

## Ограничение количества объектов

Фронтенд отображает максимум `maxObjects`.

Если backend не вернул `maxObjects`, значение берется из переменной окружения:

```env
VITE_MAX_OBJECTS_COUNT=15
```

Если переменная не задана, используется `15`.

## Runtime фронта

`src/api/monitoringApi.js` работает как адаптер реального backend: загружает snapshot через `GET /api/objects` и подписывается на события через `VITE_API_EVENTS_ENDPOINT`.

## Время в интерфейсе

Бэкенд присылает только абсолютное время `updatedAt`.

Подписи вида `только что`, `5 сек назад`, `1 мин назад` фронтенд считает сам на основе этих полей.
