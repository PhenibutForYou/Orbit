# Сводка по API и конфигурации фронтенда

> Актуальная упрощенная версия контракта находится в `docs/api-integration-summary-v2.md`.

Документ оставлен как расширенная историческая сводка. Актуальный контракт находится в `docs/api-integration-summary-v2.md`.

Текущий фронтенд работает через реальный API-слой: `src/api/monitoringApi.js`, `src/api/historyApi.js`, `src/api/httpClient.js`.

## Общая логика работы

Фронтенд работает в два этапа:

1. При загрузке страницы получает полный снимок всех уже подключенных объектов.
2. После этого слушает события от бэкенда: создание, обновление и удаление объектов.

Важно: фронтенд не должен сам решать, аварийный объект или нет. Общий статус объекта и статус каждого параметра телеметрии приходят извне.

## Стартовый запрос

Ожидаемый endpoint:

```http
GET /api/objects
```

Ответ:

```json
{
  "objects": [],
  "maxObjects": 15
}
```

`objects` содержит все объекты, которые уже есть на сервере на момент открытия сайта. Если объектов нет, приходит пустой массив. Если на сервере уже есть подключенные объекты, они должны прийти сразу.

## Формат объекта

```json
{
  "id": "car-01",
  "type": "car",
  "name": "Машина-01",
  "description": "Сервисный автомобиль патрульной группы",
  "status": "normal",
  "coordinates": "-48.42152, 37.88403",
  "updatedAt": "2026-07-02T12:45:31.000Z",
  "static": [
    { "label": "Точка начала", "value": "-71.42015, 62.14577" },
    { "label": "Точка прибытия", "value": "64.30184, -32.86719" }
  ],
  "telemetry": []
}
```

Обязательные поля:

- `id` - уникальный идентификатор объекта.
- `type` - машинный тип объекта.
- `name` - название объекта.
- `description` - описание объекта.
- `status` - общий статус объекта.
- `coordinates` - текущие координаты объекта на карте.
- `updatedAt` - ISO-время последнего обновления объекта.
- `static` - статические параметры объекта.
- `telemetry` - динамические параметры объекта.

## Типы объектов

Сейчас используются четыре типа:

- `car` - машина.
- `warehouse` - склад.
- `fuel-station` - АЗС.
- `drone` - дрон.

Фронт технически может принять и другие типы, если для них будут переданы корректные поля, иконка/отображение будут дополнительно согласованы.

## Координаты

Координаты используются не географические, а условные декартовы координаты карты:

- центр карты - `0, 0`;
- диапазон по X - от `-100` до `100`;
- диапазон по Y - от `-100` до `100`;
- формат строки - `"x, y"`;
- дробные значения ожидаются, например `"51.42152, -17.62080"`.

Любой объект может изменить координаты в любой момент. У объекта нет поля `movable`. Если бэкенд прислал новые `coordinates`, фронт плавно переместит маркер.

## Статусы

Допустимые значения статуса:

- `normal` - норма, зеленый цвет.
- `warning` - предупреждение, желтый цвет.
- `alert` - критическое состояние, красный цвет.
- `nodata` - нет данных, серый цвет.

Статус объекта приходит в поле `status`.

Статус отдельного параметра телеметрии приходит в поле `telemetry[].status`.

Если от сущности долго нет данных, это должен определить бэкенд и прислать `status: "nodata"`.

## Телеметрия

Пример параметра:

```json
{
  "key": "fuel",
  "label": "Топливо",
  "value": "72 %",
  "rawValue": 72,
  "unit": "%",
  "range": { "min": 0, "max": 100 },
  "progress": 72,
  "status": "normal",
  "updatedAt": "2026-07-02T12:45:31.000Z"
}
```

Поля параметра:

- `key` - стабильный ключ параметра внутри объекта.
- `label` - название параметра в интерфейсе.
- `value` - готовое отображаемое значение.
- `rawValue` - числовое значение.
- `unit` - единица измерения.
- `range` - допустимый диапазон.
- `progress` - значение шкалы от `0` до `100`.
- `status` - статус конкретного параметра.
- `updatedAt` - время обновления параметра.

`key` должен быть стабильным. По нему фронт понимает, какой именно параметр обновился.

## Статические параметры

Статические параметры передаются в `static`.

Пример:

```json
[
  { "label": "Точка начала", "value": "-71.42015, 62.14577" },
  { "label": "Точка прибытия", "value": "64.30184, -32.86719" }
]
```

Важно: точки начала и прибытия считаются статическими параметрами. На карте они пока отдельно не отображаются.

## События после загрузки

После стартового snapshot фронт должен получать события. Транспорт можно выбрать позже: WebSocket, SSE или polling.

Для фронта важен формат события.

## Событие создания

```json
{
  "type": "created",
  "object": {
    "id": "drone-05",
    "type": "drone",
    "name": "Дрон-05",
    "description": "БПЛА наблюдения периметра",
    "status": "normal",
    "coordinates": "12.50124, -61.90421",
    "updatedAt": "2026-07-02T12:45:36.000Z",
    "static": [],
    "telemetry": []
  }
}
```

После `created` фронт добавляет объект в список и на карту с анимацией появления.

## Событие обновления

```json
{
  "type": "updated",
  "objectId": "drone-05",
  "patch": {
    "coordinates": "14.88441, -59.12005",
    "status": "warning",
    "updatedAt": "2026-07-02T12:45:39.000Z",
    "telemetryUpdates": [
      {
        "key": "charge",
        "label": "Заряд",
        "value": "28 %",
        "rawValue": 28,
        "unit": "%",
        "range": { "min": 0, "max": 100 },
        "progress": 28,
        "status": "warning",
        "updatedAt": "2026-07-02T12:45:39.000Z"
      }
    ]
  }
}
```

`patch` может содержать только изменившиеся поля.

Поддерживаемые изменения:

- `coordinates` - фронт плавно двигает маркер.
- `status` - фронт меняет цвет объекта.
- `updatedAt` - фронт обновляет время последнего изменения.
- `telemetryUpdates` - фронт обновляет только указанные параметры.

При обновлении телеметрии:

- строка измененного параметра подсвечивается;
- прогрессбар плавно переходит от старого значения к новому;
- при простом клике по объекту подсветка не запускается.

## Событие удаления

```json
{
  "type": "deleted",
  "objectId": "drone-05"
}
```

После `deleted` фронт проигрывает анимацию удаления. Если удаленный объект был выбран, выделение снимается.

## Ограничение количества объектов

Фронт отображает максимум `maxObjects`.

Сейчас значение по умолчанию:

```txt
15
```

В реальном ответе `GET /api/objects` желательно возвращать:

```json
{
  "maxObjects": 15
}
```

## Конфиги API

```env
VITE_API_BASE_URL=
VITE_API_OBJECTS_ENDPOINT=/api/objects
VITE_API_EVENTS_ENDPOINT=/api/objects/events
VITE_API_HISTORY_ENDPOINT=/api/history
VITE_API_HISTORY_EXPORT_ENDPOINT=/api/history/export
VITE_API_REALTIME_TRANSPORT=sse
VITE_API_REQUEST_TIMEOUT_MS=10000
VITE_MAX_OBJECTS_COUNT=15
```

Описание:

- `VITE_MAX_OBJECTS_COUNT` - максимальное количество объектов во фронте.
- `VITE_API_BASE_URL` - origin backend, если он отличается от фронта.
- `VITE_API_REALTIME_TRANSPORT` - транспорт realtime: `sse` или `ws`.
- `VITE_API_REQUEST_TIMEOUT_MS` - timeout HTTP-запросов.

## Где это подключено во фронте

Основные файлы:

- `src/api/monitoringApi.js` - API-адаптер реального backend для snapshot и realtime-событий.
- `src/state/ObjectsContext.jsx` - загрузка snapshot, подписка на события, применение изменений.
- `src/domain/InfrastructureObject.js` - доменная модель объекта.
- `src/components/map/MapPanel.jsx` - карта и список маркеров.
- `src/components/map/MapMarker.jsx` - маркер объекта.
- `src/components/monitoring/ObjectDetails.jsx` - правая панель объекта.
- `src/components/monitoring/TelemetryMetric.jsx` - отображение параметра телеметрии.
- `src/utils/constants.js` - конфиги демо и лимит объектов.
- `src/utils/status.js` - подписи и логика отображения статусов.
- `src/utils/coordinates.js` - преобразование координат в позицию на карте.
- `src/utils/time.js` - форматирование времени.

## Что должен сделать бэкенд

Минимально:

1. Реализовать `GET /api/objects`.
2. Отдавать объекты в формате из этого документа.
3. Давать способ получать события `created`, `updated`, `deleted`.
4. Присылать общий статус объекта.
5. Присылать статус каждого параметра телеметрии.
6. Присылать новые координаты при их изменении.
7. Присылать `nodata`, если данных от сущности давно нет.
8. Соблюдать лимит объектов или отдавать `maxObjects`, чтобы фронт понимал ограничение.

## Что фронт считает сам

Фронт сам считает только отображение:

- русскую подпись статуса;
- относительное время вида `только что`, `5 сек назад`, `1 мин назад`;
- позицию маркера на карте из координат;
- визуальные анимации появления, удаления, перемещения и обновления.

Фронт не считает аварийность и не решает, какой статус должен быть у объекта.

## Архив

Архив пока не входит в текущий monitoring API. Планируемая логика:

```http
GET /api/history
GET /api/history/export
```

Архив будет дорабатываться отдельным этапом.
